关于Vue中的sync修饰符以及样式穿透/deep/

发表于 2020-12-15 |

记录vue中.sync中的用法。在平常的UI组件库中都需要用到对话框，但是有时候需要封装一些功能。然后每次都需要写一个<a-modal></a-modal>这样的标签然后给visible这个布尔变量，父组件跟子组件都需要能够操作这个变量，并且能够修改。但是在Vue中不允许子组件能够修改父组件的值。平常我个人使用的方法是给子组件去传递一个对象在对象里去写属性，然后子组件去修改的话，可以响应，因为是对象引用的地址，所以没什么太大的问题，但是有时候会出现引用丢失的情况，再加上我的watch会操作很多的监听，导致有时候饮用关系需要去深度监听，对性能很不好。所有除了这种有没有更方便的方法。
第一种：

/deep/

/deep/的主要作用就是在用一些UI库的时候去覆盖样式的，在vue中写样式的话一般都会加上scoped属性，不会影响到全局的样式，但是有时候仅仅是一个很小的样式如果要覆盖样式的话要么你取一个特定的ID名然后写成全局的样式，或者直接加入全局样式表里，前提是不影响其他的项目。一般我个人倾向前面的，好跟踪也方便维护。那么怎么在scoped里面写权重比较高的css样式呢。还是推荐先看官网怎么说的。写法：
.app-fff .app-title /deep/ span { margin-left:0px } 这是正常的写法但是我把/deep/写到前面最前面，不生效后面也不生效，正常的话是生效的，可能是其他问题导致的 还有就是可以简写为>>>，其实这个本质是在class上加入独一无二的hash值。
